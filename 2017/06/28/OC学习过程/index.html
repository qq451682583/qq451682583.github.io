语法基础

创建/销毁OC对象

  • 关键字加“@”区分c/c++

  • 基类NSObject 类似Java的 Object

  • 单继承 与java一致

  • 接口（oc叫协议）：@protocol

  • 多继承，基于协议 。 类似java

  • 支持多态，抽象

  • 虚函数 所有函数都是虚函数

  • 异常处理 和java类似

  • 布尔类型 BOOL YES,NO两种

  • id类型 o-c中，每个目标都可以表达为id类型，范性，可以认为是NSObject 或者 void

  • nil nil等同于null，表示一个目标的指针

  • OC类分为2个文件， 一个是.h 一个是.m文件 .h 文件存放类和函数的申明 .m文件存放类的具体实现

  • 类申明使用关键字 @interface @end 来申明

  • 类的实现使用关键字 @implementation @end 来实现

  • oc用 #import 来表示包含头文件格式 。编译器自动处理避免重复包含的操作

  • oc函数全部为public ,”:”为函数的一部分

  • oc不是严格的函数重载

类声明（接口）

Objective-C中，调用方法被称为发送消息 ，通知对象执行某种操作。语法如下：[shape draw]

  1. 前面的短线/- 表示这是objective-c方法的声明，用来区分函数原型（C语言中）与（Objective—C中）方法声明的方式。短线后面是方法的返回类型/比如(void) ，位于圆括号中。

    • 注意，方法的声明在括号后面，@end之前 {}区域里只有变量的定义，这点和C++很不一样。

  2. 返回类型后面自然是函数名字，和c语言一样的，不同的是参数声明方法
